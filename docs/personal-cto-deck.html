Why nobody's built this

1.
AI coding just crossed the threshold
Claude Opus/Sonnet can now write production code autonomously with tool use. This was impossible 12 months ago.
2.
Orchestration is the hard part
Single-agent wrappers are easy. Coordinating N parallel agents with git isolation, budget limits, and human gates is an engineering challenge.
3.
Trust requires domain depth
You can't build a generic "AI engineer" that actually works. You need deep integration with specific workflows (Salesforce, SOC 2, your CI/CD).
4.
Big companies move slowly
GitHub and Microsoft are iterating on copilots. The jump from "autocomplete" to "autonomous team" is a fundamentally different product.
